# Auth Pages UI Skill

This skill generates authentication-related interfaces for SaaS applications.

Examples:

- Login
- Register
- Forgot password
- Reset password
- Onboarding welcome pages

---

# UX Goals

Authentication flows must be:

- simple
- fast
- distraction-free

Users should complete the action in seconds.

---

# Layout

Prefer centered layouts with a strong visual hierarchy.

Typical structure:

Logo or product name

Page title

Form fields

Primary action button

Secondary links

Optional illustration or decorative background.

---

# Visual Style

Auth pages can be slightly more expressive than dashboards.

Allowed elements:

- subtle gradients
- background textures
- decorative shapes
- soft shadows

But the form itself must remain clear and readable.

---

# Form Structure

Typical login form:

- email
- password
- remember me (optional)

Include:

Forgot password link.

Register page should include:

- name
- email
- password
- confirm password.

---

# Component Usage

Use shadcn components:

- Card
- Input
- Button
- Label
- Form

Keep forms clean and readable.

---

# Responsiveness

Forms should work well on mobile and desktop.

Centered layouts are recommended.
